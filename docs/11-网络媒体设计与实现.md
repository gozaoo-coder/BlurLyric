# 网络媒体（Resource Manager）设计与实现

## 概述

网络媒体（Resource Manager）是 BlurLyric 3.0 的资源管理核心模块，负责大文件资源（专辑图片、音乐文件）的获取、管理与释放，提供统一的资源管理方案。

## 设计原则

1. **引用计数**：资源自动管理生命周期，防止内存泄漏
2. **LRU 缓存**：使用 LRU 策略管理资源池
3. **统一接口**：所有资源类型提供一致的获取和释放接口
4. **Trace 驱动**：通过 Trace 机制统一获取资源
5. **性能优先**：优先使用本地/缓存资源，减少网络请求

## 核心架构

### 1. 资源基类（Resource）

`Resource` 是所有资源的基类，实现引用计数和自动释放：

```javascript
class Resource {
    // 引用计数管理
    retain()      // 增加引用计数
    release()     // 减少引用计数，计数为 0 时销毁
    destroy()     // 立即销毁资源
    
    // 属性
    objectURL     // 资源的 ObjectURL
    refCount     // 当前引用计数
}
```

### 2. 资源池（ResourcePool）

`ResourcePool` 实现 LRU 缓存策略管理资源：

```javascript
class ResourcePool {
    // 最大资源数限制
    // LRU 淘汰策略
    // 资源自动清理
}
```

### 3. 数据模型

#### 3.1 Track（单曲数据模型）

支持 Trace 来源追踪的单曲数据模型：

**主要属性：**
- `id`, `name`, `src`, `lyric` - 基本信息
- `artists`, `album` - 关联数据
- `traces` - 所有来源 Trace 列表
- `primaryTrace` - 主来源 Trace

**主要方法：**
```javascript
// 来源检查
hasLocalResource()      // 检查是否有本地资源
hasMultipleTraces()  // 检查是否有多个来源
getTraceBySourceType(type)  // 根据类型获取 Trace

// 资源获取
getAlbumCover(apiAdapter, resolution)
getMusicFile(apiAdapter)
fetchResourceByTrace(sourceManager)

// 导航
navigateToArtist(index)
navigateToAlbum()

// 生命周期
releaseResources()
```

#### 3.2 Artist（艺术家数据模型）

```javascript
class Artist {
    id, name, alias, avatarUrl, bio
    traces, primaryTrace
    
    getAvatar(apiAdapter, resolution)
    navigateToSource(sourceManager)
}
```

#### 3.3 Album（专辑数据模型）

```javascript
class Album {
    id, name, picUrl, year, artists
    traces, primaryTrace
    
    getCover(apiAdapter, resolution)
    navigateToSource(sourceManager)
    navigateToArtist(index)
}
```

#### 3.4 TrackList（歌曲列表）

```javascript
class TrackList {
    tracks, currentIndex, currentTrack
    
    next(), prev()
    addTrack(), removeTrack()
    shuffle(), clear()
}
```

### 4. 资源类型

#### 4.1 AlbumCoverResource（专辑封面资源）

```javascript
class AlbumCoverResource {
    constructor(albumId, resolution)
    async load(apiAdapter)
    release()
}
```

#### 4.2 MusicFileResource（音乐文件资源）

```javascript
class MusicFileResource {
    constructor(songId)
    async load(apiAdapter)
    release()
}
```

### 5. 资源获取器（ResourceFetcher）

`ResourceFetcher` 提供统一的资源获取流程：

**核心流程：**
1. 检查是否为本地资源
2. 检查缓存是否存在
3. 如果缓存存在，直接返回缓存
4. 如果缓存不存在，从网络下载
5. 将下载的资源存入缓存
6. 返回资源 URL

**主要方法：**
```javascript
// 获取资源（自动缓存）
async fetch(trace, options)

// 检查资源是否已缓存
async isCached(trace)

// 获取缓存路径
async getCachedPath(trace)

// 预加载资源（后台下载）
async preload(trace)

// 批量预加载
async preloadBatch(traces, concurrency)
```

## 使用示例

### 创建数据模型

```javascript
import { Track, Artist, Album, TrackList } from '@/api/resources';

// 从原始数据创建 Track
const track = Track.fromRaw(trackData);

// 创建数组
const tracks = Track.fromRawArray(rawTracks);

// 创建 TrackList
const trackList = new TrackList(tracks);
```

### 使用 Trace 来源追踪

```javascript
// 获取所有 Trace
const traces = track.traces;

// 获取主 Trace
const primaryTrace = track.primaryTrace;

// 检查是否有本地资源
if (track.hasLocalResource()) {
    console.log('有本地资源');
}

// 根据来源类型获取 Trace
const apiTrace = track.getTraceBySourceType('api');
```

### 获取资源

```javascript
// 获取专辑封面
const coverResource = await track.getAlbumCover(sourceManager, 368);
const coverURL = coverResource.objectURL;

// 获取音乐文件
const musicResource = await track.getMusicFile(sourceManager);
const musicURL = musicResource.objectURL;

// 释放资源
coverResource.release();
musicResource.release();
```

### 通过 ResourceFetcher 获取资源

```javascript
import { getResourceFetcher } from '@/api/resourceFetcher';

const fetcher = getResourceFetcher(sourceManager);

// 获取资源
const result = await fetcher.fetch(trace);
// result: { objectURL, destroyObjectURL, fromCache }

// 检查缓存
const isCached = await fetcher.isCached(trace);

// 预加载
await fetcher.preload(trace);

// 批量预加载
await fetcher.preloadBatch(traces, 3);
```

### 导航到关联数据

```javascript
// 跳转到艺人
const artist = track.navigateToArtist(0);
if (artist) {
    const artistDetail = await artist.navigateToSource(sourceManager);
}

// 跳转到专辑
const album = track.navigateToAlbum();
if (album) {
    const albumDetail = await album.navigateToSource(sourceManager);
}
```

### 使用 TrackList

```javascript
const trackList = new TrackList(tracks);

// 播放控制
trackList.setCurrentIndex(0);
const currentTrack = trackList.currentTrack;
const nextTrack = trackList.next();
const prevTrack = trackList.prev();

// 编辑
trackList.addTrack(newTrack);
trackList.removeTrack(5);
trackList.shuffle();

// 清理
trackList.clear();
```

## 资源生命周期管理

### 引用计数

```javascript
const resource = await track.getAlbumCover(sourceManager);

// 增加引用
resource.retain();

// 减少引用（计数为 0 时自动销毁
resource.release();
```

### 自动释放

```javascript
// 组件卸载时释放所有资源
track.releaseResources();
```

## 扩展开发

### 自定义资源类型

```javascript
import { Resource } from '@/api/resources';

class CustomResource {
    #resource;
    
    constructor(id) {
        this.#resource = null;
    }
    
    async load(apiAdapter) {
        if (this.#resource) {
            this.#resource.retain();
            return this.#resource;
        }
        // 加载资源...
    }
    
    release() {
        if (this.#resource) {
            this.#resource.release();
        }
    }
}
```

## 架构优势

1. **内存安全**：引用计数确保资源正确释放
2. **性能优化**：缓存机制减少重复加载
3. **统一接口**：所有资源类型使用一致的 API
4. **来源追踪**：完整的 Trace 机制支持多源数据
5. **易于扩展**：新增资源类型只需遵循接口

## 资源获取流程

```
用户请求
    ↓
检查 Trace 类型
    ↓
是本地资源？ → 是 → 直接读取本地文件
    ↓ 否
检查缓存 → 缓存命中？ → 是 → 返回缓存资源
    ↓ 否
从网络下载
    ↓
存入缓存
    ↓
返回 ObjectURL
```
