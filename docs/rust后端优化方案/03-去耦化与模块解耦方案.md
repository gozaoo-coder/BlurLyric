# 去耦化与模块解耦方案

## 1. 概述

本文档针对 BlurLyric-3.0 Rust 后端代码中发现的重复代码和耦合问题，提出系统性的去耦化和模块解耦方案。通过提取公共函数、引入 Repository 模式、实现依赖注入，提升代码的可维护性、可测试性和可扩展性。

## 2. 问题分析

### 2.1 重复代码统计

| 函数/逻辑 | 重复位置 | 重复次数 | 影响范围 |
|-----------|---------|---------|---------|
| `current_timestamp()` | incremental_scanner.rs, music_library_cache.rs, resource_cache.rs, performance_monitor.rs, lib.rs 内联 | **5处** | 全局时间戳获取 |
| `is_music_file()` | lib.rs:273-278, incremental_scanner.rs:192-197 | **2处** (完全相同) | 音乐文件过滤 |
| `normalize_for_dedup()` / `normalize_string()` | lib.rs:353-367, music_deduplicator.rs:47-59 | **2处** (差1个中文逗号) | 字符串标准化 |
| `calculate_audio_quality_score()` | lib.rs:405-443, lib.rs:846-885, music_library_cache.rs:102-136, music_deduplicator.rs:275-313 | **4处** (几乎相同) | 音质评分计算 |
| 缓存目录获取 | lib.rs get_cache_dir, music_library_cache.rs get_library_cache_dir, resource_cache.rs get_resource_cache_dir | **3处** (相似) | 缓存路径管理 |

### 2.2 耦合问题

- **全局状态耦合**: 各模块直接依赖全局 `Mutex` 静态变量
- **硬编码依赖**: 模块间通过函数调用直接耦合，无法独立测试
- **数据访问分散**: 歌曲查询逻辑散布在 lib.rs 多个函数中

### 3.1 创建 common/utils.rs 模块

在 `src-tauri/src/common/` 目录下创建统一的工具函数模块：

```rust
// src-tauri/src/common/utils.rs
// 公共工具函数 - 统一管理跨模块复用的工具函数

use std::fs::DirEntry;
use std::path::Path;
use std::time::{SystemTime, UNIX_EPOCH};

/// 获取当前 Unix 时间戳（秒）
///
/// 用于统一各模块的时间戳生成逻辑，避免重复实现。
/// 返回自 UNIX_EPOCH 以来的秒数。
pub fn current_timestamp() -> u64 {
    SystemTime::now()
        .duration_since(UNIX_EPOCH)
        .unwrap_or_default()
        .as_secs()
}

/// 检查目录条目是否为支持的音乐文件格式
///
/// 支持的格式: mp3, ogg, flac, m4a, wav, aac
///
/// # Arguments
/// * `entry` - 目录条目引用
///
/// # Returns
/// 如果是支持的音乐文件格式返回 true，否则返回 false
pub fn is_music_file(entry: &DirEntry) -> bool {
    matches!(
        entry.path().extension().and_then(|ext| ext.to_str()),
        Some("mp3" | "ogg" | "flac" | "m4a" | "wav" | "aac")
    )
}

/// 标准化字符串用于去重比较
///
/// 将字符串转换为小写，去除空格、下划线、连字符、引号、括号等字符，
/// 以及中英文逗号，用于生成一致的指纹进行比较。
///
/// # Arguments
/// * `s` - 需要标准化的字符串
///
/// # Returns
/// 标准化后的字符串
pub fn normalize_for_dedup(s: &str) -> String {
    s.to_lowercase()
        .trim()
        .replace(" ", "")
        .replace("_", "")
        .replace("-", "")
        .replace("'", "")
        .replace("\"", "")
        .replace("(", "")
        .replace(")", "")
        .replace("[", "")
        .replace("]", "")
        .replace("，", "")  // 中文逗号
        .replace(",", "")   // 英文逗号
}

/// 计算音频质量评分
///
/// 基于比特率、音频格式、采样率和时长计算综合质量分数，
/// 用于在去重合并时选择最佳音质的音轨。
///
/// # Arguments
/// * `bitrate` - 比特率 (kbps)，可选
/// * `format` - 音频格式字符串 (如 "flac", "mp3")
/// * `sample_rate` - 采样率 (Hz)，可选
/// * `duration` - 音频时长 (秒)，可选
///
/// # Returns
/// 质量评分分数 (0-1400+)
pub fn calculate_audio_quality_score(
    bitrate: Option<u32>,
    format: &str,
    sample_rate: Option<u32>,
    duration: Option<f64>,
) -> u32 {
    let mut score = 0u32;

    // 比特率分数 (最高320分)
    if let Some(bitrate) = bitrate {
        score += bitrate.min(320);
    }

    // 格式分数 (无损格式优先)
    score += match format.to_lowercase().as_str() {
        "flac" => 500,
        "wav" | "aiff" => 400,
        "aac" | "m4a" => 300,
        "mp3" => 200,
        "ogg" => 250,
        "wma" => 150,
        _ => 100,
    };

    // 采样率分数 (最高480分，48kHz)
    if let Some(sample_rate) = sample_rate {
        score += (sample_rate / 100).min(480);
    }

    // 时长分数 (完整曲目优先，最高100分)
    if let Some(duration) = duration {
        if duration > 180.0 {  // 超过3分钟
            score += 100;
        } else if duration > 60.0 {  // 超过1分钟
            score += 50;
        }
    }

    score
}

/// 获取应用缓存根目录
///
/// 返回系统缓存目录下的 com.blurlyric.app 子目录。
/// 如果目录不存在会自动创建。
///
/// # Returns
/// 成功返回缓存目录路径，失败返回错误信息
pub fn get_app_cache_dir() -> Result<std::path::PathBuf, String> {
    let path = dirs::cache_dir().ok_or("Cannot get cache directory")?;
    let mut path = path;
    path.push("com.blurlyric.app");
    if !path.exists() {
        std::fs::create_dir_all(&path).map_err(|e| e.to_string())?;
    }
    Ok(path)
}

/// 获取音乐库缓存子目录
///
/// # Returns
/// 音乐库缓存目录路径
pub fn get_library_cache_subdir() -> std::path::PathBuf {
    get_app_cache_dir()
        .map(|p| p.join("library"))
        .unwrap_or_default()
}

/// 获取资源缓存子目录
///
/// # Returns
/// 资源缓存目录路径
pub fn get_resource_cache_subdir() -> std::path::PathBuf {
    get_app_cache_dir()
        .map(|p| p.join("resource_cache"))
        .unwrap_or_default()
}
```

## 4. Repository 模式设计

### 4.1 设计目标

将数据访问逻辑从业务逻辑中分离出来：
- 定义抽象的数据访问接口 (`MusicRepository` trait)
- 当前实现封装现有的内存存储 (`InMemoryMusicRepository`)
- 未来可无缝切换到持久化存储 (`SqliteMusicRepository`)
- 提高可测试性：单元测试时可注入 Mock Repository

### 4.2 Trait 定义

```rust
// src-tauri/src/repository/music_repository.rs
//! 音乐数据仓库抽象层
//!
//! 定义音乐数据的 CRUD 操作接口，支持多种后端实现。

use async_trait::async_trait;
use serde::{Deserialize, Serialize};
use std::collections::HashMap;
use std::path::PathBuf;

/// 歌曲基本信息 (轻量级，用于列表展示)
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct SongSummary {
    pub id: u32,
    pub name: String,
    pub artists: Vec<String>,
    pub album: Option<String>,
    pub duration: Option<f64>,
    pub path: PathBuf,
}

/// 歌曲详细信息 (完整信息)
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct SongDetail {
    #[serde(flatten)]
    pub summary: SongSummary,
    pub track_number: u16,
    pub genre: Option<String>,
    pub year: Option<u32>,
    pub bitrate: Option<u32>,
    pub sample_rate: Option<u32>,
    pub quality_score: u32,
}

/// 艺术家信息
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ArtistInfo {
    pub id: u32,
    pub name: String,
    pub song_count: usize,
}

/// 专辑信息
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct AlbumInfo {
    pub id: u32,
    pub name: String,
    pub artists: Vec<String>,
    pub song_count: usize,
}

/// 查询条件
#[derive(Debug, Default)]
pub struct SongQuery {
    pub keyword: Option<String>,       // 关键词搜索 (歌名/艺术家/专辑)
    pub artist_id: Option<u32>,        // 按艺术家筛选
    pub album_id: Option<u32>,         // 按专辑筛选
    pub min_duration: Option<f64>,     // 最小时长
    pub max_duration: Option<f64>,     // 最大时长
    pub offset: Option<usize>,         // 分页偏移
    pub limit: Option<usize>,          // 分页大小
}

/// 音乐仓库 trait
#[async_trait]
pub trait MusicRepository: Send + Sync {
    /// 获取所有歌曲摘要列表
    async fn list_songs(&self, query: Option<SongQuery>) -> Result<Vec<SongSummary>, String>;

    /// 根据 ID 获取歌曲详情
    async fn get_song_by_id(&self, id: u32) -> Result<Option<SongDetail>, String>;

    /// 根据 ID 获取歌曲文件路径
    async fn get_song_path_by_id(&self, id: u32) -> Result<Option<PathBuf>, String>;

    /// 获取所有艺术家
    async fn list_artists(&self) -> Result<Vec<ArtistInfo>, String>;

    /// 获取指定艺术家的歌曲
    async fn get_songs_by_artist(&self, artist_id: u32) -> Result<Vec<SongDetail>, String>;

    /// 获取所有专辑
    async fn list_albums(&self) -> Result<Vec<AlbumInfo>, String>;

    /// 获取指定专辑的歌曲
    async fn get_songs_by_album(&self, album_id: u32) -> Result<Vec<SongDetail>, String>;

    /// 搜索歌曲
    async fn search_songs(&self, keyword: &str) -> Result<Vec<SongSummary>, String>;

    /// 获取歌曲总数
    async fn count_songs(&self) -> Result<usize, String>;

    /// 批量添加或更新歌曲
    async fn upsert_songs(&self, songs: Vec<SongDetail>) -> Result<(), String>;

    /// 删除指定路径的歌曲
    async fn remove_song_by_path(&self, path: &PathBuf) -> Result<bool, String>;
}
```

### 4.3 InMemoryRepository 实现

```rust
// src-tauri/src/repository/in_memory_repository.rs
//! 内存实现的 MusicRepository
//!
//! 封装现有的全局状态，提供 Repository 接口的内存实现。

use super::music_repository::*;
use crate::common::utils::calculate_audio_quality_score;
use parking_lot::RwLock;
use std::collections::HashMap;
use std::sync::Arc;

/// 内存音乐仓库
pub struct InMemoryMusicRepository {
    /// 歌曲存储 (id -> SongDetail)
    songs: RwLock<HashMap<u32, SongDetail>>,
    /// 艺术家存储 (id -> ArtistInfo)
    artists: RwLock<HashMap<u32, ArtistInfo>>,
    /// 专辑存储 (id -> AlbumInfo)
    albums: RwLock<HashMap<u32, AlbumInfo>>,
    /// 辅助索引: artist_id -> Vec<song_id>
    artist_songs_index: RwLock<HashMap<u32, Vec<u32>>>,
    /// 辅助索引: album_id -> Vec<song_id>
    album_songs_index: RwLock<HashMap<u32, Vec<u32>>>,
    /// 路径索引: path_str -> song_id
    path_index: RwLock<HashMap<String, u32>>,
}

impl InMemoryMusicRepository {
    /// 创建新的内存仓库实例
    pub fn new() -> Self {
        Self {
            songs: RwLock::new(HashMap::new()),
            artists: RwLock::new(HashMap::new()),
            albums: RwLock::new(HashMap::new()),
            artist_songs_index: RwLock::new(HashMap::new()),
            album_songs_index: RwLock::new(HashMap::new()),
            path_index: RwLock::new(HashMap::new()),
        }
    }

    /// 从现有数据批量初始化 (迁移用)
    pub fn initialize_from_legacy(
        songs: HashMap<u32, SongDetail>,
        artists: HashMap<u32, ArtistInfo>,
        albums: HashMap<u32, AlbumInfo>,
    ) -> Self {
        let mut repo = Self::new();

        // 构建索引
        let mut artist_songs: HashMap<u32, Vec<u32>> = HashMap::new();
        let mut album_songs: HashMap<u32, Vec<u32>> = HashMap::new();
        let mut path_idx: HashMap<String, u32> = HashMap::new();

        for (&id, song) in &songs {
            // TODO: 从 song 提取 artist_id 和 album_id 并构建索引
            path_idx.insert(song.summary.path.display().to_string(), id);
        }

        *repo.artist_songs_index.write() = artist_songs;
        *repo.album_songs_index.write() = album_songs;
        *repo.path_index.write() = path_idx;
        *repo.songs.write() = songs;
        *repo.artists.write() = artists;
        *repo.albums.write() = albums;

        repo
    }
}

#[async_trait]
impl MusicRepository for InMemoryMusicRepository {
    async fn list_songs(&self, query: Option<SongQuery>) -> Result<Vec<SongSummary>, String> {
        let songs = self.songs.read();
        let summaries: Vec<SongSummary> = songs.values()
            .map(|s| s.summary.clone())
            .collect();
        Ok(summaries)
    }

    async fn get_song_by_id(&self, id: u32) -> Result<Option<SongDetail>, String> {
        let songs = self.songs.read();
        Ok(songs.get(&id).cloned())
    }

    async fn get_song_path_by_id(&self, id: u32) -> Result<Option<PathBuf>, String> {
        let songs = self.songs.read();
        Ok(songs.get(&id).map(|s| s.summary.path.clone()))
    }

    async fn list_artists(&self) -> Result<Vec<ArtistInfo>, String> {
        let artists = self.artists.read();
        Ok(artists.values().cloned().collect())
    }

    async fn get_songs_by_artist(&self, artist_id: u32) -> Result<Vec<SongDetail>, String> {
        let index = self.artist_songs_index.read();
        let song_ids = index.get(&artist_id).cloned().unwrap_or_default();
        drop(index);  // 释放读锁

        let songs = self.songs.read();
        let result: Vec<SongDetail> = song_ids.iter()
            .filter_map(|id| songs.get(id).cloned())
            .collect();
        Ok(result)
    }

    async fn list_albums(&self) -> Result<Vec<AlbumInfo>, String> {
        let albums = self.albums.read();
        Ok(albums.values().cloned().collect())
    }

    async fn get_songs_by_album(&self, album_id: u32) -> Result<Vec<SongDetail>, String> {
        let index = self.album_songs_index.read();
        let song_ids = index.get(&album_id).cloned().unwrap_or_default();
        drop(index);

        let songs = self.songs.read();
        let result: Vec<SongDetail> = song_ids.iter()
            .filter_map(|id| songs.get(id).cloned())
            .collect();
        Ok(result)
    }

    async fn search_songs(&self, keyword: &str) -> Result<Vec<SongSummary>, String> {
        let keyword_lower = keyword.to_lowercase();
        let songs = self.songs.read();
        let results: Vec<SongSummary> = songs.values()
            .filter(|s| {
                s.summary.name.to_lowercase().contains(&keyword_lower)
                    || s.summary.artists.iter().any(|a|
                        a.to_lowercase().contains(&keyword_lower))
                    || s.album.as_ref()
                        .map(|al| al.to_lowercase().contains(&keyword_lower))
                        .unwrap_or(false)
            })
            .map(|s| s.summary.clone())
            .collect();
        Ok(results)
    }

    async fn count_songs(&self) -> Result<usize, String> {
        let songs = self.songs.read();
        Ok(songs.len())
    }

    async fn upsert_songs(&self, songs: Vec<SongDetail>) -> Result<(), String> {
        let mut store = self.songs.write();
        let mut path_idx = self.path_index.write();

        for song in songs {
            let id = song.summary.id;
            path_idx.insert(song.summary.path.display().to_string(), id);
            store.insert(id, song);
        }
        Ok(())
    }

    async fn remove_song_by_path(&self, path: &PathBuf) -> Result<bool, String> {
        let path_str = path.display().to_string();
        let path_idx = self.path_index.read();

        if let Some(&id) = path_idx.get(&path_str) {
            drop(path_idx);
            let mut songs = self.songs.write();
            let removed = songs.remove(&id).is_some();
            drop(songs);

            if removed {
                let mut path_idx = self.path_index.write();
                path_idx.remove(&path_str);
                return Ok(true);
            }
        }
        Ok(false)
    }
}
```

## 6. 迁移计划

### 6.1 第一阶段: 提取公共函数 (低风险)

**目标**: 消除明显的代码重复

**步骤**:
1. 创建 `common/utils.rs` 模块
2. 实现 6 个公共函数
3. 逐模块修改导入，删除本地重复定义
4. 运行现有测试确保无回归

**预计工作量**: 2-3 小时
**风险等级**: 低 (纯重构，不改变行为)

### 6.2 第二阶段: 引入 Repository 接口 (中等风险)

**目标**: 解耦数据访问层

**步骤**:
1. 定义 `MusicRepository` trait
2. 实现 `InMemoryMusicRepository`
3. 逐步将 lib.rs 中的命令函数改为使用 Repository
4. 构建辅助索引优化查询性能

**预计工作量**: 1-2 天
**风险等级**: 中 (需确保接口设计合理)

### 6.3 第三阶段: 依赖注入改造 (较高风险)

**目标**: 实现完整的 DI 架构

**步骤**:
1. 创建 `AppState` 结构
2. 修改 `run()` 函数注册 State
3. 重构所有 Tauri Command 签名
4. 更新前端调用 (如有变化)

**预计工作量**: 2-3 天
**风险等级**: 中高 (涉及大量函数签名变更)

### 6.4 第四阶段: 清理与优化 (持续)

**目标**: 移除遗留的全局状态

**步骤**:
1. 移除不再使用的全局 `Mutex` 变量
2. 清理废弃的类型别名
3. 补充单元测试和集成测试
4. 性能基准测试

## 10. 总结

本方案通过三个层次的改进实现系统性去耦:

1. **函数级去耦**: 提取 6 个公共函数到 `common/utils.rs`，消除 ~200 行重复代码
2. **接口级解耦**: 引入 `MusicRepository` trait，将数据访问抽象化
3. **架构级解耦**: 通过 Tauri State 机制实现依赖注入，彻底解除对全局状态的硬编码依赖

这种渐进式的重构策略可以在不影响现有功能的前提下，显著提升代码质量和架构健壮性。
