## 1. 现状分析

### 1.1 当前架构问题

当前音乐播放功能全部耦合在 [App.vue](../src/App.vue) 中（约 **630+ 行**音乐相关代码），存在以下问题：

| 问题类型 | 具体表现 |
|---------|---------|
| **上帝组件** | App.vue 承担了播放器核心、列表管理、状态管理、事件处理等全部职责 |
| **强耦合** | 9 个子组件通过 `inject` 直接依赖 App.vue 的方法/状态 |
| **无事件系统** | 组件间通过 provide/inject 硬绑定，无法解耦 |
| **无法测试** | 播放逻辑嵌入 Vue 组件，无法独立测试 |
| **Trace 未集成** | 资源获取仍使用旧的 `manager.tauri.getMusicFile()` 方式，未走 Trace 统一接口 |

### 3.2 播放列表管理 (`playlist/`)

#### 3.2.1 用户操作类 — `PlaylistManager`

继承/组合 `PlaylistOperations`，提供基础 CRUD 操作：

```javascript
// playlist/PlaylistOperations.js - 操作接口定义
export class PlaylistOperations {
    #tracks = [];

    replace(tracks, startIndex = 0)     // 替换整个列表 → 触发 PLAYLIST_REPLACE
    push(track)                          // 追加单曲 → 触发 PLAYLIST_PUSH
    pushBatch(tracks)                    // 批量追加
    remove(index)                        // 按索引移除
    clear()                              // 清空列表
    insert(track, index)                 // 指定位置插入
    move(fromIndex, toIndex)             // 移动排序

    get tracks() { return [...this.#tracks]; }
    get length() { return this.#tracks.length; }
    get isEmpty() { return this.#tracks.length === 0; }
    get(index) { return this.#tracks[index] ?? null; }
}
```

#### 3.2.2 智能推荐类 — `SmartPlaylist`

用于需要动态增长的场景（如私人FM）：

```javascript
// playlist/SmartPlaylist.js - 可增长列表抽象
export class SmartPlaylist extends PlaylistOperations {
    #growFunction;      // 增长方法: () => Promise<Track[]> | Promise<Track>
    #minBufferCount;    // 最小缓冲数量
    #isLoading = false;

    constructor(options = {}) {
        super();
        this.#growFunction = options.growFunction;
        this.#minBufferCount = options.minBufferCount ?? 5;
    }

    async checkAndGrow() {
        if (this.length < this.#minBufferCount && !this.#isLoading) {
            this.#isLoading = true;
            const newTracks = await this.#growFunction();
            if (Array.isArray(newTracks)) {
                this.pushBatch(newTracks);
            } else {
                this.push(newTracks);
            }
            this.#isLoading = false;
        }
    }

    getCurrentOrGrow(currentIndex) {
        const track = this.get(currentIndex);
        if (!track || currentIndex >= this.length - 2) {
            this.checkAndGrow();  // 异步预加载，不阻塞
        }
        return track;
    }
}
```

**使用示例**:

```javascript
// 创建私人FM智能列表
const fmPlaylist = new SmartPlaylist({
    growFunction: async () => {
        const result = await apiSource.getPersonalFmTracks(10);
        return result.map(raw => Track.fromRaw(raw));
    },
    minBufferCount: 5
});

// 设置到播放器
player.setPlaylist(fmPlaylist);
// 当播放到末尾时，自动调用 growFunction 获取更多歌曲
```

### 3.3 音频引擎 (`core/AudioEngine.js`)

**设计原则**: 默认使用原生 DOM Audio，支持替换为其他实现（如 Howler.js、Web Audio API 等）。

```javascript
// core/AudioEngine.js - 音频引擎基类
export class AudioEngine {
    #audioElement;
    #destroyURLFn;
    #isActive = true;

    constructor() {
        this.#audioElement = document.createElement('audio');
    }

    get audioDom() { return this.#audioElement; }
    get currentTime() { return this.#audioElement.currentTime; }
    get duration() { return this.#audioElement.duration; }
    get paused() { return this.#audioElement.paused; }
    get volume() { return this.#audioElement.volume; }
    set volume(v) { this.#audioElement.volume = Math.max(0, Math.min(1, v)); }

    async loadFromTrace(trace) {
        const { objectURL, destroyObjectURL } = await trace.fetchResource();
        this.#audioElement.src = objectURL;
        this.#destroyURLFn = destroyObjectURL;
    }

    async loadFromURL(url) {
        this.#audioElement.src = url;
        this.#destroyURLFn = null;
    }

    play() { return this.#audioElement.play(); }
    pause() { this.#audioElement.pause(); }
    seek(time) { this.#audioElement.currentTime = time; }

    on(event, handler) { this.#audioElement.addEventListener(event, handler); }
    off(event, handler) { this.#audioElement.removeEventListener(event, handler); }

    destroy() {
        this.#isActive = false;
        this.#audioElement.pause();
        if (this.#destroyURLFn) this.#destroyURLFn();
        this.#audioElement.src = '';
        this.#audioElement.remove();
    }

    get isActive() { return this.#isActive; }
}
```

### 3.4 播放模式管理 (`core/PlayMode.js`)

```javascript
// core/PlayMode.js - 播放模式管理器
export const PlayMode = {
    LOOP_PLAYLIST: 'loopPlaylist',       // 列表循环
    LOOP_SINGLE: 'loopSingle',           // 单曲循环
    STOP_AFTER_SINGLE: 'stopAfterSingle',// 播完本曲停止
    RANDOM_PLAY: 'randomPlay',           // 随机播放
    REVERSE_LOOP: 'reverseLoop',         // 倒序循环（新增）
};

export const PlayModeLabels = {
    [PlayMode.LOOP_PLAYLIST]: '列表循环',
    [PlayMode.LOOP_SINGLE]: '单曲循环',
    [PlayMode.STOP_AFTER_SINGLE]: '播完本曲暂停',
    [PlayMode.RANDOM_PLAY]: '随机播放',
    [PlayMode.REVERSE_LOOP]: '倒序循环',
};

export class PlayModeManager {
    #currentMode = PlayMode.LOOP_PLAYLIST;
    #allModes = Object.values(PlayMode);

    get current() { return this.#currentMode; }
    get label() { return PlayModeLabels[this.#currentMode]; }
    get allModes() { return [...this.#allModes]; }

    next() {
        const idx = this.#allModes.indexOf(this.#currentMode);
        this.#currentMode = this.#allModes[(idx + 1) % this.#allModes.length];
        return this.#currentMode;
    }

    set(mode) {
        if (this.#allModes.includes(mode)) {
            this.#currentMode = mode;
        }
    }

    getNextIndex(currentIndex, playlistLength) {
        switch (this.#currentMode) {
            case PlayMode.LOOP_PLAYLIST:
                return (currentIndex + 1) % playlistLength;
            case PlayMode.LOOP_SINGLE:
                return currentIndex;
            case PlayMode.STOP_AFTER_SINGLE:
                return (currentIndex + 1) % playlistLength;
            case PlayMode.RANDOM_PLAY:
                let rnd;
                do { rnd = Math.floor(Math.random() * playlistLength); } while (rnd === currentIndex && playlistLength > 1);
                return rnd;
            case PlayMode.REVERSE_LOOP:
                return currentIndex <= 0 ? playlistLength - 1 : currentIndex - 1;
            default:
                return 0;
        }
    }

    getPrevIndex(currentIndex, playlistLength) {
        switch (this.#currentMode) {
            case PlayMode.LOOP_PLAYLIST:
                return currentIndex <= 0 ? playlistLength - 1 : currentIndex - 1;
            case PlayMode.REVERSE_LOOP:
                return (currentIndex + 1) % playlistLength;
            case PlayMode.RANDOM_PLAY:
                let rnd;
                do { rnd = Math.floor(Math.random() * playlistLength); } while (rnd === currentIndex && playlistLength > 1);
                return rnd;
            default:
                return currentIndex <= 0 ? playlistLength - 1 : currentIndex - 1;
        }
    }

    shouldAutoAdvance() {
        return this.#currentMode !== PlayMode.STOP_AFTER_SINGLE;
    }
}
```

### 3.5 切歌策略 (`transition/`)

```javascript
// transition/TransitionStrategy.js - 策略接口
export class TransitionStrategy {
    async execute(player, targetIndex, options = {}) {
        throw new Error('Must implement execute()');
    }
}

// transition/TransitionNextMusic.js - 平滑过渡（默认策略）
export class TransitionNextMusic extends TransitionStrategy {
    async execute(player, targetIndex, options = {}) {
        const oldEngine = player.audioEngine;
        const nextTrack = player.playlist.get(targetIndex);
        const leastTime = options.leastTime ?? this.calcRemainingTime(oldEngine);

        player.switchToIndex(targetIndex);
        await player.loadCurrentTrack();
        const newEngine = player.audioEngine;

        newEngine.volume = player.state.volume * 0.5;
        player.play();

        anime({ targets: newEngine, duration: leastTime, volume: player.state.volume, easing: 'linear' });
        anime({
            targets: oldEngine,
            duration: leastTime,
            volume: 0,
            easing: 'linear',
            complete: () => { oldEngine.destroy(); }
        });
    }

    calcRemainingTime(engine) {
        return 1000 * (engine.duration - engine.currentTime);
    }
}

// transition/InstantSwitch.js - 即时切换
export class InstantSwitch extends TransitionStrategy {
    async execute(player, targetIndex) {
        player.audioEngine?.destroy();
        player.switchToIndex(targetIndex);
        await player.loadCurrentTrack();
        player.play();
    }
}
```

```javascript
import { reactive, computed } from 'vue';

export function createPlayerState() {
    const state = reactive({
        playing: false,
        loading: false,
        canplay: false,
        error: null,
        currentTime: 0,

}
```

## 4. Trace 资源集成方案

### 4.1 统一资源获取接口

播放器通过 Trace 的 `fetchResource()` 方法统一获取音频资源：

```
用户点击播放
    ↓
MusicPlayer.loadCurrentTrack()
    ↓
获取当前 Track 对象
    ↓
track.primaryTrace  →  Trace 对象
    ↓
trace.fetchResource()
    ↓  (内部根据 fetchMethod.type 分发)
    ├─ LOCAL_FILE  → invoke('get_music_file') → Blob URL
    ├─ DOWNLOAD   → fetch(url) → 缓存 → Blob URL
    ├─ STREAM     → 直接返回 URL
    ├─ API_CALL   → POST endpoint → Blob URL
    ↓
返回 { objectURL, destroyObjectURL, fromCache }
    ↓
AudioEngine.audioDom.src = objectURL
```

### 4.2 与现有 Trace 类的对接

现有 [trace.js](../src/api/source/trace.js) 已实现完整的 `fetchResource()` 方法（含缓存），直接复用：

- `trace.fetchResource({ useCache: true })` → `{ objectURL, destroyObjectURL, fromCache }`
- 无需修改任何 Trace 相关代码
- 播放器只需调用此单一接口

## 5. App.vue 改造方案

### 5.1 改造前后的 App.vue 对比

**改造前 (App.vue ~880行)**:
- 包含所有播放器逻辑 (~630行)
- data 中有 audioManager, musicTrack, musicTrackIndex, audioState, trackState
- methods 中有 15+ 个播放器方法
- provide/inject 暴露 20+ 个播放器属性/方法

**改造后 (App.vue)**:
- 仅保留 `createPlayer()` 调用和 `provide(player)` 单一注入
- 所有播放器逻辑迁移至 `src/module/musicPlayer/`
- 通过事件总线解耦子组件依赖

### 5.2 改造后 App.vue 核心结构

```javascript
// App.vue 改造后（仅展示关键部分）
import { createPlayer } from '@/module/musicPlayer';

export default {
    data() {
        return {
            player: null,          // 替代原来的 audioManager + 所有状态
            // ... 其他非播放器状态保持不变
        };
    },

    async mounted() {
        this.player = await createPlayer({
            config: this.config.audio,
        });
        this.player.events.on(PlayerEvents.TRACK_CHANGE, this.handleTrackChange);
        this.player.events.on(PlayerEvents.STATE_CHANGE, this.handleStateChange);
    },

    provide() {
        return {
            player: computed(() => this.player),  // 唯一的播放器注入点
            // ... 其他非播放器的 provide 保持不变
        };
    },

    beforeUnmount() {
        this.player?.destroy();
    }
};
```

### 5.3 子组件迁移方式

**旧写法**:
```javascript
// musicInfoPage.vue
inject: ['currentMusicInfo', 'audioState', 'audioManager', 'nextMusic', 'prevMusic', ...]
// 模板中: @click="prevMusic()"  : {{ audioState.currentTime }}
```

**新写法**:
```javascript
// musicInfoPage.vue
import { PlayerEvents } from '@/module/musicPlayer/events/PlayerEvents';
inject: ['player']
// 模板中: @click="player.prev()"  : {{ player.state.currentTime }}
```

## 7. 关键设计决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 状态管理 | Vue reactive + 自带状态对象 | 复用现有 Vue 生态，无需引入 Pinia |
| 事件系统 | 自研 EventBus | 轻量级，不引入外部依赖 |
| 音频引擎 | DOM Audio 默认 + 可替换接口 | 兼容性最好，预留扩展能力 |
| 列表管理 | 组合模式（普通 + 智能） | 满足固定列表和动态增长两种场景 |
| 切歌策略 | 策略模式 | 易扩展新的过渡效果 |
| 资源获取 | Trace.fetchResource() 统一接口 | 符合接口规范，自动缓存 |

## 8. 风险与注意事项

1. **anime.js 依赖**: `transitionNextMusic` 使用了 anime.js 做音量渐变，新模块需要确认该依赖可用
2. **MediaSession API**: 系统媒体会话（锁屏控制）需要在播放器内重新接入
3. **向后兼容**: 迁移期间需要保证旧 inject 接口仍可用（或一次性全量替换）
4. **ObjectURL 内存管理**: 切歌时必须正确销毁旧的 ObjectURL，避免内存泄漏
5. **Tauri invoke 调用**: 本地文件获取仍通过 Tauri IPC，Trace.fetchLocalFile 内部已处理
