## 3. 关键流程实现

### 3.1 扫描与合并流程
- StorageSource.fullRescan() 扫描本地文件
- 构建 Song/Album/Artist 实体
- 提交至 LibraryManager
- MergeStrategy 判定是否为相同实体
- 相同则追加 Trace，不同则创建新实体

### 3.2 APISource 资源获取流程
- 前端通过 IPC 调用 SourceManager.getAPISourceMusicFile()
- 检查 pendingRequests 防重复守卫
- 查询 LibraryManager 本地是否已存在资源
- 不存在则调用 APISource.fetch*() 获取远程资源
- 写入 TempStorageSource 缓存
- 返回资源给前端

### 3.3 偏好保存流程
- 用户点击"喜欢"
- FavorSystem.addFavorite()
- 检查 LibraryManager 中该 Song 的来源
- 根据来源情况执行不同的保存策略

### 3.4 实体清理与悬空引用处理流程
- Source 注销或 TempStorageSource 启动清空
- LibraryManager 移除对应 Trace
- 检查实体 traces 是否为空
- 为空则触发 on_entity_cleanup 事件
- FavorSystem 处理悬空引用
