 三、Source 继承体系
════════════════════════════════════════════════════════════════════════════════

  Source（资源来源 · 基类）
  │
  │  通用接口（所有子类必须实现）：
  │  ├── getSongFile( trace )       → 返回音频文件数据
  │  ├── getSongLyric( trace )      → 返回歌词文本
  │  ├── getAlbumPicture( trace )   → 返回专辑图片数据
  │  └── sourceID / sourceType      → 身份标识
  │
  ├── StorageSource（存储类来源 · 继承 Source）
  │   │
  │   ├── ① 资源获取 / 变动处理
  │   │   ├── fullRescan()                        // 完全重新扫描（必须实现）
  │   │   └── watchFileChanges() → Option<Watcher> // 文件变更监听（可选实现）
  │   │
  │   ├── ② 查询接口
  │   │   ├── getAllSongs()    → Vec<Song>
  │   │   ├── getAllAlbums()   → Vec<Album>
  │   │   ├── getAllArtists()  → Vec<Artist>
  │   │   └── search( keyword ) → SearchResult
  │   │
  │   ├── ③ 提交与注销
  │   │   ├── 扫描后向 LibraryManager 提交实体 + Trace
  │   │   ├── 变动时注销已提交的 Trace
  │   │   └── 迁移/路径变更时 → 主动更新已提交 Trace 的 details.path
  │   │
  │   ├── ④ 基类方法实现
  │   │   ├── getSongFile( trace )     → 读取 trace.details.path 本地文件
  │   │   ├── getSongLyric( trace )    → 从本地文件元数据提取歌词
  │   │   └── getAlbumPicture( trace ) → 从本地文件元数据提取专辑图片
  │   │
  │   └── 内置存储实现
  │       │
  │       ├── LocalStorageSource（本地存储 · 默认启用）
  │       │   ├── 用户可自定义多个需扫描/监听的音乐文件夹
  │       │   ├── 实现资源文件变更监听
  │       │   ├── 提供完整的资源文件夹迁移方法
  │       │   └── 迁移后主动更新 LibraryManager 中的 Trace.details.path
  │       │
  │       ├── TempStorageSource（临时缓存存储 · 默认启用）
  │       │   ├── 缓存来自 APISource 的临时音乐文件
  │       │   ├── 支持写入与删除临时缓存
  │       │   ├── 超过 1GB 时自动清理内部文件
  │       │   ├── 程序关闭时注销内部文件
  │       │   ├── 启动时注册为空 → LibraryManager 清理其残留 Trace
  │       │   └── 基于 LocalStorageSource 实现
  │       │
  │       ├── FavorStorageSource（收藏存储 · 默认启用）
  │       │   ├── 收藏来自 APISource 的音乐（由 FavorSystem 驱动写入）
  │       │   ├── 允许用户自定义存储路径
  │       │   ├── 提供迁移方法（迁移后更新 Trace.details.path）
  │       │   └── 基于 LocalStorageSource 实现
  │       │
  │       └── WebDevSource（Web 开发源 · 基于 StorageSource 实现）
  │
  └── APISource（API 类来源 · 继承 Source，与 StorageSource 平行独立）
      │
      ├── 查询接口
      │   ├── searchSong( keyword )    → Vec<Song>
      │   ├── searchAlbum( keyword )   → Vec<Album>
      │   ├── searchArtist( keyword )  → Vec<Artist>
      │   ├── getSongDetail( trace )   → SongDetail
      │   ├── getAlbumDetail( trace )  → AlbumDetail
      │   └── getArtistDetail( trace ) → ArtistDetail
      │
      ├── 私有方法（各实例按自身 API 规范实现）
      │   ├── fetchSongFile( trace )     → Binary   // 写入 Temp/FavorStorageSource
      │   ├── fetchAlbumPicture( trace ) → Binary   // 写入音乐文件元数据
      │   └── fetchSongLyric( trace )    → String   // 写入音乐文件元数据
      │
      ├── 基类方法实现（内部调用私有方法）
      │   ├── getSongFile( trace )     → fetchSongFile( trace )
      │   ├── getSongLyric( trace )    → fetchSongLyric( trace )
      │   └── getAlbumPicture( trace ) → fetchAlbumPicture( trace )
      │
      └── 衍生能力
          │
          ├── api_server_module（API 服务器模块）
          │   │

          │   │   └── 将自身数据以 APISource 协议对外暴露
          │   │
          │   └── 技术栈：Tauri + Vite + Vue + Rust backend
          │
          └── StorageSource → APIServer 桥接
