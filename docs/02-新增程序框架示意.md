# BlurLyric 新增程序框架示意文档

## 一、框架设计目标

基于手绘框架图，新增框架旨在实现以下目标：

1. **统一资源管理**：整合本地、网络、WebDav等多种音乐来源
2. **灵活配置系统**：支持用户自定义配置和偏好管理
3. **高效缓存策略**：优化临时文件和网络资源的缓存管理
4. **模块化设计**：各模块职责清晰，便于扩展和维护

### 3.3 元数据(MusicTag)扫描模块

#### 3.3.1 模块职责

| 属性 | 说明 |
|------|------|
| **核心职责** | 音乐文件的元数据提取、统计与存储 |
| **输入** | 音乐文件路径 |
| **输出** | 标准化的MusicTag数据 |
| **统计内容** | 歌曲数量、专辑数量、艺术家数量等 |

#### 3.3.2 功能范围

- **音频文件元数据解析**：支持ID3、Vorbis Comment等标签格式
- **音乐指纹生成**：使用BLAKE3算法生成文件指纹
- **去重处理**：自动识别和合并重复音轨
- **扫描结果持久化**：JSON格式存储扫描结果

#### 3.3.3 扫描流程

```
开始扫描
    │
    ▼
读取缓存指纹数据
    │
    ▼
遍历文件系统
    │
    ▼
检查文件修改时间
    │
    ├── 未修改 ──► 使用缓存数据
    │
    └── 已修改 ──► 重新解析元数据
                        │
                        ▼
                  生成文件指纹
                        │
                        ▼
                  检测重复文件
                        │
                        ▼
                  更新缓存数据
    │
    ▼
保存扫描结果
    │
    ▼
更新统计信息
```

### 3.6 临时区模块

#### 3.6.1 模块职责

| 属性 | 说明 |
|------|------|
| **核心职责** | 管理临时文件的存储与清理 |
| **存储内容** | 单曲文件、专辑图片 |
| **清理策略** | 达上限即删除（LRU或FIFO） |
| **简单信息** | 不缓存（如文字信息、显示详情） |

#### 3.6.2 功能范围

- **临时文件缓存管理**：下载的网络资源临时存储
- **缓存上限控制**：防止磁盘空间无限增长
- **自动清理策略**：LRU（最近最少使用）或FIFO（先进先出）
- **用户偏好缓存**：用户添加偏好时的详细缓存

#### 3.6.3 缓存策略

```
┌─────────────────────────────────────────────────────────────┐
│                        临时区缓存                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  缓存内容：                                                  │
│  ├── 单曲文件 (MP3/FLAC等)                                   │
│  ├── 专辑图片 (WebP/JPG)                                     │
│  └── 歌词文件 (LRC/BLF)                                      │
│                                                             │
│  不缓存内容：                                                │
│  ├── 文字信息（歌曲名、艺人名等）                             │
│  └── 显示详情（元数据、统计信息）                             │
│                                                             │
│  清理策略：                                                  │
│  ├── 容量上限：可配置（默认500MB）                            │
│  ├── 时间上限：可配置（默认7天）                              │
│  └── 算法：LRU（最近最少使用）                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 关键交互流程

#### 流程1：音乐扫描流程

```
用户设置目录 ──► 本地音乐处理 ──► 元数据扫描 ──► 资源来源管理器
                                              │
                                              ▼
                                    更新本机存储(统计与存储)
                                              │
                                              ▼
                                    更新配置区(扫描结果)
```

**详细步骤**：
1. 用户在配置管理UI中添加本地音乐目录
2. 本地音乐处理模块遍历文件系统
3. 元数据扫描模块解析音乐文件标签
4. 生成文件指纹并检测重复
5. 更新本地存储的缓存数据
6. 更新配置区的扫描结果统计

#### 流程2：网络资源获取流程

```
用户请求 ──► 资源来源管理器 ──► 网络媒体来源
                                  │
                    ┌─────────────┼─────────────┐
                    ▼             ▼             ▼
               baseUrl1      baseUrl2      baseUrl3
                    │             │             │
                    └─────────────┴─────────────┘
                                  │
                                  ▼
                         返回网络来源类
                                  │
                    ┌─────────────┼─────────────┐
                    ▼             ▼             ▼
                搜索结果      音乐详情        歌词获取
                    │             │             │
                    └─────────────┴─────────────┘
                                  │
                                  ▼
                         临时区(缓存单曲/专辑图)
                                  │
                                  ▼
                         用户偏好管理(可选)
```

**详细步骤**：
1. 用户发起搜索或播放请求
2. 资源来源管理器路由到网络来源
3. 网络来源尝试多个baseUrl
4. 获取搜索结果或音乐详情
5. 下载文件到临时区缓存
6. 可选：添加到用户偏好（如"我喜欢的音乐"）

#### 流程3：播放流程

```
用户播放请求 ──► 资源来源管理器 ──► 判断来源类型
                                      │
                        ┌─────────────┼─────────────┐
                        ▼             ▼             ▼
                      本地资源      网络资源      临时缓存
                        │             │             │
                        ▼             ▼             ▼
                   直接播放      下载到临时区    直接使用
                                      │
                                      ▼
                                更新最近播放
                                      │
                                      ▼
                                用户偏好管理
```

**详细步骤**：
1. 用户点击播放按钮
2. 资源来源管理器判断音乐来源类型
3. 本地资源：直接读取文件播放
4. 网络资源：先下载到临时区，再播放
5. 临时缓存：直接使用缓存文件
6. 更新播放历史和统计

## 七、框架特点总结

### 7.1 核心设计亮点

1. **统一资源抽象**：通过Source基类统一不同来源的资源访问
2. **多来源支持**：本地、网络、WebDav等多种来源无缝切换
3. **智能缓存策略**：三级缓存 + LRU清理策略
4. **灵活配置系统**：支持多baseUrl、优先级管理
5. **用户偏好管理**：歌单、收藏、播放历史完整支持

### 7.2 与现有架构的整合

新增框架与现有BlurLyric架构的整合点：

| 现有模块 | 整合方式 |
|----------|----------|
| `Source`基类 | 扩展支持网络来源 |
| `cache_manager.rs` | 扩展支持临时区管理 |
| `App.vue`状态 | 增加用户偏好状态 |
| `router` | 增加配置管理页面 |
| `components` | 增加配置管理UI组件 |

### 7.3 技术实现建议

1. **后端优先**：先实现Rust后端的配置管理和缓存模块
2. **渐进式前端**：逐步替换现有组件，保持兼容性
3. **数据迁移**：提供配置和数据迁移工具
4. **测试覆盖**：为核心模块编写单元测试和集成测试
