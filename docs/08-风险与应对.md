# 08 - 风险与应对

## 8.1 风险管理概述

本文档识别 BlurLyric-3.0 项目开发过程中可能面临的风险，并制定相应的应对措施。

### 8.1.1 风险等级定义

| 等级 | 可能性 | 影响 | 说明 |
|------|--------|------|------|
| 高 | > 70% | 严重 | 需立即制定应对方案 |
| 中 | 30%-70% | 中等 | 需要监控和预案 |
| 低 | < 30% | 轻微 | 记录并观察 |

### 8.1.2 风险分类

| 分类 | 说明 |
|------|------|
| 技术风险 | 技术实现、架构设计相关风险 |
| 进度风险 | 项目延期、资源不足风险 |
| 质量风险 | 代码质量、性能问题风险 |
| 外部风险 | 第三方依赖、API变更风险 |
| 安全风险 | 数据安全、隐私保护风险 |

## 8.2 技术风险

### 8.2.1 元数据解析兼容性问题

| 属性 | 内容 |
|------|------|
| 风险ID | TEC-001 |
| 风险描述 | 不同音频格式的元数据标准不一致，可能导致解析失败或数据丢失 |
| 可能性 | 中 |
| 影响 | 高 |
| 风险等级 | **高** |

**风险分析**:
- ID3v2 有多个版本（2.3, 2.4），字段定义有差异
- Vorbis Comment 字段命名不统一
- 部分文件可能使用非标准编码
- 损坏的标签可能导致解析崩溃

**应对措施**:

| 措施 | 说明 | 状态 |
|------|------|------|
| 使用成熟解析库 | 使用 Symphonia 库处理多种格式 | ✅ 已实施 |
| 降级处理策略 | 解析失败时使用文件名作为默认值 | ✅ 已实施 |
| 编码自动检测 | 支持多种字符编码自动转换 | ✅ 已实施 |
| 错误容忍机制 | 跳过损坏的标签，继续解析其他内容 | ✅ 已实施 |

**应急预案**:
```rust
// 解析失败时的降级处理
fn parse_music_file(file: PathBuf) -> Result<Song, String> {
    match parser.parse(&file) {
        Ok(metadata) => { /* 使用解析结果 */ },
        Err(_) => {
            // 降级：使用文件名和默认值
            Song {
                name: file_name,
                ar: vec![get_or_create_artist("Unknown Artist")],
                al: get_or_create_album("Unknown Album"),
                // ...
            }
        }
    }
}
```

| 可能性 | 中 |
| 影响 | 中 |
| 风险等级 | **中** |

**风险分析**:
- 部分设备可能不支持 WGPU
- GPU 驱动版本差异
- 显存不足可能导致处理失败

**应对措施**:

| 措施 | 说明 | 状态 |
|------|------|------|

| CPU降级处理 | GPU不可用时自动切换到CPU | ✅ 已实施 |
| 资源限制 | 限制并发GPU任务数量 | ✅ 已实施 |

**应急预案**:
```rust
pub async fn resize_with_gpu_fallback(
    image_data: Vec<u8>,
    max_resolution: u32,
) -> Result<Vec<u8>, String> {
    // 优先尝试GPU处理
    if let Ok(gpu_result) = try_gpu_resize(&image_data, max_resolution).await {
        return Ok(gpu_result);
    }
    
    // 降级到CPU处理
    cpu_resize(&image_data, max_resolution)
}
```

**应急预案**:
```javascript
// 多源降级策略
class SourceManager {
    async searchAll(keyword, options) {
        const results = await Promise.allSettled(
            this.sources.map(source => source.searchAll(keyword, options))
        );
        
        // 过滤成功的结果
        const successResults = results
            .filter(r => r.status === 'fulfilled')
            .map(r => r.value);
        
        if (successResults.length === 0) {
            throw new Error('所有数据源均不可用');
        }
        
        return this.mergeResults(successResults);
    }
}
```

## 8.6 合规性风险

### 8.6.1 商标侵权风险

| 属性 | 内容 |
|------|------|
| 风险ID | COM-001 |
| 风险描述 | 硬编码具体音乐平台名称可能导致商标侵权，收到律师函 |
| 可能性 | 高 |
| 影响 | 高 |
| 风险等级 | **高** |

**风险分析**:
- 在代码中硬编码平台名称（如"网易云音乐"、"QQ音乐"）可能侵犯商标权
- 枚举类型中包含平台名称存在法律风险
- 开源项目更容易被发现和追责

**应对措施**:

| 措施 | 说明 | 状态 |
|------|------|------|
| 移除硬编码平台名称 | SourceType.Api 不包含具体平台 | ✅ 已实施 |
| 用户自定义标识 | 通过 sourceId 和 baseUrl 配置 | ✅ 已实施 |
| 接口标准化 | 符合规范即可接入，无需修改代码 | ✅ 已实施 |
| 用户责任声明 | 用户自行负责所接入API的合法性 | 🔄 规划中 |

**合规设计**:

```rust
// ❌ 错误设计（有风险）
enum ApiType {
    Netease,    // 网易云音乐 - 商标侵权风险
    QQMusic,    // QQ音乐 - 商标侵权风险
    Kugou,      // 酷狗音乐 - 商标侵权风险
}

// ✅ 正确设计（合规）
enum SourceType {
    Storage(StorageType),  // Local 或 WebDAV
    Api,                   // API型（不硬编码具体平台）
}

struct Trace {
    source_id: String,     // 用户自定义标识
    source_name: String,   // 用户自定义显示名称
    base_url: Option<String>, // API基础地址
    // ...
}
```
