# BlurLyric 3.0 当前程序框架文档

## 一、项目概述

**BlurLyric 3.0** 是一个基于 **Tauri + Vue 3** 构建的跨平台桌面音乐播放器，采用前后端分离的架构设计，支持本地音乐库管理和网络音乐资源获取。

### 1.1 技术栈

| 层级 | 技术栈 | 版本 | 用途 |
|------|--------|------|------|
| **前端框架** | Vue 3 | 3.x | UI渲染和状态管理 |
| **桌面框架** | Tauri | 2.0 | 跨平台桌面应用封装 |
| **后端语言** | Rust | 1.70+ | 系统级操作和性能密集型任务 |
| **构建工具** | Vite | 5.x | 前端开发和构建 |
| **路由管理** | Vue Router | 4.x | 单页应用路由 |
| **动画库** | Anime.js | - | 流畅的UI动画效果 |
| **UI图标** | Bootstrap Icons | - | 图标系统 |

### 1.2 项目结构

```
BlurLyric-3.0/
├── docs/                       # 项目文档
├── public/                     # 静态资源
├── src/                        # 前端源代码

├── package.json                # Node.js依赖配置
├── vite.config.js              # Vite构建配置
```


### 2.3 状态管理

项目使用 **Vue 3 Provide/Inject API** 进行跨组件状态共享：

**核心状态** (定义在 `App.vue`):
```javascript
provide() {
    return {
        // 音乐相关
        currentMusicInfo: computed(() => this.musicTrack[this.musicTrackIndex]),
        musicTrack: computed(() => this.musicTrack),
        musicTrackIndex: computed(() => this.musicTrackIndex),
        
        // 播放控制
        audioManager: computed(() => this.audioManager),
        audioState: computed(() => this.audioState),
        trackState: computed(() => this.trackState),
        
        // 数据源
        source: computed(() => this.source),
        
        // 方法
        pushMusic: this.pushMusic,
        nextMusic: this.nextMusic,
        prevMusic: this.prevMusic,
    };
}
```

**状态说明**:
| 状态 | 类型 | 说明 |
|------|------|------|
| `musicTrack` | Array | 当前播放列表 |
| `musicTrackIndex` | Number | 当前播放索引 |
| `audioManager` | Object | 音频管理器实例 |
| `audioState` | Object | 音频状态（播放/暂停/进度等） |
| `trackState` | Object | 音轨状态（播放模式等） |
| `source` | Object | 当前数据源 |

### 3.2 核心模块详解

#### 3.2.1 音乐库管理 (lib.rs)

**核心数据结构**:
```rust
struct Song {
    name: String,                   // 歌曲名称
    id: u32,                        // 唯一ID
    ar: Vec<Artist>,                // 艺人列表
    al: Album,                      // 专辑信息
    src: PathBuf,                   // 文件路径
    duration: Option<f64>,          // 时长
    bitrate: Option<u32>,           // 比特率
    sample_rate: Option<u32>,       // 采样率
    sources: Vec<TrackSourceInfo>,  // 多来源支持
    primary_source_index: usize,    // 主来源索引
}

struct Artist {
    name: String,
    id: u32,
}

struct Album {
    name: String,
    id: u32,
    pic: Option<String>,            // 封面路径
}
```

**核心功能**:
| 功能 | 说明 |
|------|------|
| 音乐扫描 | 递归扫描指定目录中的音乐文件 |
| 元数据解析 | 读取音乐标签（ID3、Vorbis Comment等） |
| 去重合并 | 根据指纹自动合并重复音轨 |
| 音质选择 | 自动选择最高音质的音源 |

**支持格式**:
- MP3
- FLAC
- OGG
- M4A
- WAV
- AAC

#### 3.2.2 缓存管理器 (cache_manager.rs)

**职责**：管理音乐库元数据的缓存

**核心功能**:
| 功能 | 说明 |
|------|------|
| 文件指纹 | 使用 BLAKE3 哈希算法生成文件指纹 |
| 增量扫描 | 只扫描新增或修改的文件 |
| 持久化缓存 | JSON格式存储在系统缓存目录 |
| 缓存统计 | 提供缓存大小、歌曲数量等统计 |

**缓存文件**:
- `music_library_cache.json`: 音乐库元数据缓存
- `album_*.webp`: 专辑封面图片缓存

#### 3.2.3 增量扫描器 (incremental_scanner.rs)

**职责**：实现高效的增量音乐库扫描

**工作流程**:
1. 读取缓存的指纹数据
2. 对比文件修改时间
3. 只扫描新增或修改的文件
4. 更新缓存数据

#### 3.2.4 音乐去重器 (music_deduplicator.rs)

**职责**：自动识别和合并重复音乐文件

**去重策略**:
- 基于音频指纹匹配
- 保留最高音质版本
- 合并多来源信息

#### 3.2.5 图片处理器

**CPU图片处理** (`image_processor.rs`):
- 专辑封面提取
- 图片缩放（Lanczos3算法）
- 格式转换

**GPU图片处理** (`gpu_image_processor.rs`):
- 使用 wgpu 进行硬件加速
- WGSL着色器
- 高性能图片处理

#### 3.2.6 音乐标签模块 (src/music_tag/)

**与前端共享设计**:

### 4.2 缓存体系

**三级缓存架构**:

```
┌─────────────────────────────────────────────────────────────┐
│                      L1: 内存缓存                            │
│                    (Rust后端)                                │
├─────────────────────────────────────────────────────────────┤

│  ARTIST_CACHE: HashMap<u32, Artist>                         │

└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      L2: 前端缓存                            │
│                    (JavaScript)                              │
├─────────────────────────────────────────────────────────────┤

│  objectURLCache: Blob URL缓存                                │
│  lazyLoader: 懒加载缓存                                      │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      L3: 磁盘缓存                            │
│                    (文件系统)                                 │
```

## 八、当前架构总结

### 8.1 架构优势

1. **现代化技术栈**: Tauri + Vue 3 + Rust，兼顾效率和性能
2. **清晰的模块划分**: 前端/后端/存储三层架构
3. **灵活的数据源**: Source抽象支持多种数据来源
4. **完善的缓存体系**: 三级缓存优化性能
5. **优秀的用户体验**: 流畅动画、响应式设计

### 8.2 待完善点

1. **网络来源支持**: 目前仅基础框架，需完善具体实现
2. **用户偏好系统**: 歌单管理、收藏等功能待完善
3. **配置管理**: 缺少统一的配置管理UI
4. **错误处理**: 需要更完善的错误处理机制
5. **插件系统**: 支持第三方扩展的插件架构
