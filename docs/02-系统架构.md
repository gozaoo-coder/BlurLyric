# 02 - 系统架构

## 2.1 整体架构设计

BlurLyric-3.0 采用分层架构设计，结合 Tauri 框架实现前后端分离：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                           表现层 (Presentation Layer)                     │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐  ┌─────────────────┐ │
│  │  Vue.js UI  │  │  路由管理    │  │  状态管理    │  │  组件系统        │ │
│  └─────────────┘  └─────────────┘  └─────────────┘  └─────────────────┘ │
└──────────────────────────────────┬──────────────────────────────────────┘
                                   │ IPC 通信 (Tauri)
┌──────────────────────────────────▼──────────────────────────────────────┐
│                         业务逻辑层 (Business Layer)                       │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐  ┌─────────────────┐ │
│  │ Source管理器 │  │ 资源管理器   │  │ 懒加载器     │  │ 性能监控器       │ │
│  └─────────────┘  └─────────────┘  └─────────────┘  └─────────────────┘ │
└──────────────────────────────────┬──────────────────────────────────────┘
                                   │ Tauri Commands
┌──────────────────────────────────▼──────────────────────────────────────┐
│                         服务层 (Service Layer)                           │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐  ┌─────────────────┐ │
│  │ 元数据解析   │  │ 增量扫描器   │  │ 缓存管理器   │  │ 图片处理器       │ │
│  └─────────────┘  └─────────────┘  └─────────────┘  └─────────────────┘ │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐  ┌─────────────────┐ │
│  │ 音乐去重     │  │ GPU处理器    │  │ 性能监控     │  │ 文件系统操作     │ │
│  └─────────────┘  └─────────────┘  └─────────────┘  └─────────────────┘ │

│  └─────────────┘  └─────────────┘  └─────────────┘  └─────────────────┘ │
└─────────────────────────────────────────────────────────────────────────┘
```

## 2.2 分层架构详解

### 2.2.1 表现层 (Presentation Layer)

**职责**: 用户界面渲染、用户交互处理、状态管理

**核心组件**:

| 组件 | 文件位置 | 职责 |
|------|----------|------|
| Vue.js UI | `src/components/` | UI组件渲染 |
| 路由管理 | `src/router/index.js` | 页面路由控制 |
| 页面组件 | `src/router/routers/` | 各功能页面 |
| API适配器 | `src/api/` | 前端API调用封装 |

**关键类**:

```javascript
// src/api/resources/index.js
class Track          // 单曲数据模型
class Artist         // 艺人数据模型
class Album          // 专辑数据模型
class TrackList      // 播放列表管理

// src/api/source/base.js
class Source         // 数据源抽象基类

// src/api/source/tauri.js
class TauriSource    // 本地数据源实现
```

### 2.2.2 业务逻辑层 (Business Layer)

**职责**: 业务规则实现、数据源管理、资源生命周期管理

**核心模块**:

| 模块 | 文件位置 | 职责 |
|------|----------|------|
| Source管理器 | `src/api/source/index.js` | 多数据源统一管理 |
| 资源管理器 | `src/api/resources/index.js` | 大文件资源管理 |
| 懒加载器 | `src/api/lazyLoader.js` | 按需加载优化 |
| 性能监控 | `src/api/performanceMonitor.js` | 前端性能追踪 |

### 2.2.3 服务层 (Service Layer)

**职责**: 核心业务服务、数据处理、系统集成

**核心模块**:

| 模块 | 文件位置 | 职责 |
|------|----------|------|
| 元数据解析 | `src-tauri/src/music_tag/` | 音乐文件元数据解析 |
| 增量扫描器 | `src-tauri/src/incremental_scanner.rs` | 高效增量扫描 |
| 缓存管理器 | `src-tauri/src/cache_manager.rs` | 多级缓存管理 |
| 图片处理器 | `src-tauri/src/image_processor.rs` | 封面图片处理 |
| GPU处理器 | `src-tauri/src/gpu_image_processor.rs` | GPU加速处理 |
| 音乐去重 | `src-tauri/src/music_deduplicator.rs` | 智能去重合并 |
| 性能监控 | `src-tauri/src/performance_monitor.rs` | 后端性能追踪 |

### 2.2.4 数据访问层 (Data Access Layer)

**职责**: 数据持久化、文件系统操作、缓存存储

**存储结构**:

```
缓存目录 (com.blurlyric.app/)
├── music_library_cache.json     # 音乐库缓存
├── MUSIC_DIRS.json              # 音乐目录配置
├── album_{name}_{resolution}.webp # 专辑封面缓存
└── ...
```

## 2.5 核心设计模式

### 2.5.1 数据源抽象模式

```javascript
// 抽象基类定义统一接口
class Source {
    async getMusicList() { throw new Error('must implement'); }
    async getAlbums() { throw new Error('must implement'); }
    async searchTracks(keyword) { throw new Error('must implement'); }
    // ...
}

// 本地源实现
class TauriSource extends Source {
    async getMusicList() {
        return await invoke('get_music_list');
    }
}

// 网络源实现 (规划中)
class WebSource extends Source {
    async getMusicList() {
        return await fetch('/api/tracks');
    }
}
```

### 2.5.2 资源池模式

```javascript
class ResourcePool {
    #resources = new Map();
    #maxSize = 100;
    
    get(key) { return this.#resources.get(key); }
    
    set(key, resource) {
        if (this.#resources.size >= this.#maxSize) {
            this.#evict(); // LRU淘汰
        }
        this.#resources.set(key, resource);
    }
}

class Resource {
    #refCount = 1;
    
    retain() { this.#refCount++; return this; }
    release() { 
        this.#refCount--; 
        if (this.#refCount <= 0) this.destroy();
    }
}
```

### 2.5.3 缓存策略模式

```rust
// 多级缓存架构
struct CacheManager {
    cache_dir: PathBuf,
    memory_cache: Mutex<Option<MusicLibraryCache>>,
}

impl CacheManager {
    // L1: 内存缓存
    fn get_memory_cache(&self) -> Option<MusicLibraryCache>;
    
    // L2: 磁盘缓存
    fn load_from_disk(&self) -> Result<MusicLibraryCache, String>;
    
    // 缓存同步
    fn save_to_disk(&self, cache: &MusicLibraryCache) -> Result<(), String>;
}
```

## 2.6 技术选型理由

| 技术选择 | 选型理由 |
|----------|----------|
| Tauri 2.0 | 轻量级跨平台方案，Rust后端高性能 |
| Vue.js 3 | 组合式API，更好的TypeScript支持 |
| Symphonia | 纯Rust音频解码，支持多格式 |
| WGPU | 跨平台GPU计算，图片处理加速 |
| Blake3 | 高性能哈希算法，文件指纹计算 |
| Tokio | 成熟的异步运行时，高并发支持 |
