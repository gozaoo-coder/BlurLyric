## 三、优化方案详解

### 方案 A: 激进精简方案（预计缩减 35-50%）

#### A1. 移除 symphonia，自实现音频属性解析

**原理**: 音频文件的时长/采样率/声道信息存储在文件头部的固定位置，无需完整解码器。

**自实现方案**:

| 格式 | 时长信息位置 | 解析复杂度 |
|------|-------------|-----------|
| MP3 | XING/VBR header 或帧数 × 每帧采样数/采样率 | 中（需解析 MP3 帧） |
| FLAC | STREAMINFO block (固定偏移量) | **极低**（固定 34 字节） |
| OGG/Vorbis | 第 2 页的 vorbis_comment 后的 duration | 低 |
| WAV | RIFF chunk 中的 sample rate + data size | **极低** |
| M4A/AAC | mvhd/mdhd atom | 中 |

**推荐策略**: 对 FLAC/WAV 直接自实现；对 MP3 使用轻量级 looser 库或简化版帧扫描；OGG 复用 Vorbis comment 解析逻辑。

**预期收益**: 移除 ~20 个传递依赖 crate，二进制体积减少 **8-15MB**

**代码示例 — FLAC 时长解析（约 30 行）**:
```rust
fn parse_flac_duration(data: &[u8]) -> Option<f64> {
    if data.len() < 42 || &data[0..4] != b"fLaC" { return None; }
    let block_header = data[4];
    if (block_header & 0x7F) != 0 { return None; } // 必须是 STREAMINFO
    let sample_rate = u32::from_be_bytes([data[18], data[19], data[20], data[21]]) >> 12;
    if sample_rate == 0 { return None; }
    let total_samples = ((data[28] as u64) << 24) | ((data[29] as u64) << 16)
                      | ((data[30] as u64) << 8) | (data[31] as u64);
    // 特殊值: total_samples == 0xFFFFFFFFFFFF 表示未知
    if total_samples == 0xFFFFFFFFFFFF { return None; }
    Some(total_samples as f64 / sample_rate as f64)
}
```

**代码示例 — WAV 时长解析（约 15 行）**:
```rust
fn parse_wav_duration(data: &[u8]) -> Option<f64> {
    if data.len() < 44 || &data[0..4] != b"RIFF" { return None; }
    let sample_rate = u32::from_le_bytes([data[24], data[25], data[26], data[27]]);
    if sample_rate == 0 { return None; }
    let byte_rate = u32::from_le_bytes([data[28], data[29], data[30], data[31]]);
    if byte_rate == 0 { return None; }
    let data_size = u32::from_le_bytes([data[40], data[41], data[42], data[43]]);
    Some(data_size as f64 / byte_rate as f64)
}
```

#### A2. 移除 wgpu，纯 CPU 图像处理

**现状**: [gpu_image_processor.rs](src-tauri/src/gpu_image_processor.rs) 整个模块（~430 行）用于 GPU 缩放图像，且 [image_processor.rs](src-tauri/src/image_processor.rs) 中已有完整的 CPU 降级逻辑。

**影响分析**:
- **功能影响**: 无（有 CPU 降级）
- **性能影响**: 大图缩放可能慢 2-5 倍，但对专辑封面缩放（通常 < 5MP）影响可忽略
- **代码变更**: 删除 `gpu_image_processor.rs`，修改 `image_processor.rs` 移除 GPU 分支

**预期收益**: 移除 ~15 个传递依赖 crate（含 naga shader 编译器），二进制体积减少 **10-20MB**
同时移除: `bytemuck`, `pollster`

**替代增强**: 如果需要更好的 CPU 缩放性能，可引入 `fast_image_resize` crate（~200KB）代替 wgpu（~15MB+）。

---

#### A3. 精简 tokio features

**从**:
```toml
tokio = { version = "1.41.1", features = ["full"] }
```

**改为**:
```toml
tokio = { version = "1.41.1", features = ["rt-multi-thread", "sync", "time", "fs", "macros"] }
```

**移除的功能**:
- `net` — TCP/UDP（不需要，HTTP 用 reqwest）
- `process` — 子进程管理
- `signal` — Unix 信号（Windows 无意义）
- `io-util` — AsyncRead/AsyncWrite 扩展
- `parking_lot` — 替代 std sync primitives
- `metrics` — Tokio 运行时指标
- `tracing` — 日志集成

**预期收益**: 减少编译时间和约 **2-5MB** 二进制体积

---

#### A4. 统一 lazy_static → std::sync::OnceLock / once_cell::Lazy

**变更**:
- 移除 `lazy_static` 依赖
- 将 `lazy_static!` 替换为 `once_cell::sync::Lazy`（已在依赖中）或 `std::sync::OnceLock`（Rust 1.70+）

**示例转换**:
```rust
// Before (lazy_static)
lazy_static! {
    static ref CACHE: Mutex<MusicLibraryCache> = Mutex::new(MusicLibraryCache::new());
}

// After (once_cell - 已在依赖中)
static CACHE: once_cell::sync::Mutex<MusicLibraryCache> =
    once_cell::sync::Mutex::new(MusicLibraryCache::new());

// Or (std, Rust 1.70+)
static CACHE: std::sync::OnceLock<Mutex<MusicLibraryCache>> = std::sync::OnceLock::new();
```

**预期收益**: 减少 1 个依赖，体积变化微小但代码更现代

---

#### A5. 自实现 num_cpus

**替换为**:
```rust
fn get_cpu_count() -> usize {
    std::thread::available_parallelism()
        .map(|n| n.get())
        .unwrap_or(4) // 默认回退值
}
```

`std::thread::available_parallelism()` 在 Rust 1.59+ 稳定，无需外部依赖。

**预期收益**: 移除 1 个依赖及传递依赖

---

#### A6. 替换 futures::join_all 为手写循环

**当前** ([image_processor.rs:144-154](src-tauri/src/image_processor.rs#L144-L154)):
```rust
use futures::future::join_all;
let results: Vec<_> = images.into_iter()
    .map(|(img, res)| self.resize_async(img, res))
    .collect();
futures::future::join_all(futures).await
```

**替换为**:
```rust
let mut results = Vec::with_capacity(images.len());
for (img, res) in images {
    results.push(self.resize_async(img, res).await?);
}
Ok(results)
```

**预期收益**: 移除 `futures` 依赖及其传递依赖

```toml
codegen-units = 1     # 更好的优化，但编译更慢
strip = true          # 去除调试符号
panic = "abort"       # 减少 panic 展开代码
```

### 7.2 Tauri 特定优化

在 `tauri.conf.json` 中：
```json
{
  "bundle": {
    "active": true,
    "targets": "all",
    "icon": [...]
  }
}
```
确保使用 `upx`（可执行文件压缩器）进行二次压缩。

### 7.3 Windows 特定优化

- 使用 `windows-gnu` toolchain 可产生比 `msvc` 更小的二进制
- 考虑静态链接 CRT 以避免 DLL 依赖（但会增加体积，需权衡）
