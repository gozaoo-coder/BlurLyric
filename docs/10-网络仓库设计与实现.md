# 网络仓库（Source Manager）设计与实现

## 概述

网络仓库（Source Manager）是 BlurLyric 3.0 的核心模块之一，负责统一管理多个数据源（本地文件、网络 API、WebDAV 等），提供一致的接口供上层调用。

## 设计原则

1. **插件化架构**：通过工厂模式支持多种数据源类型的动态注册和扩展
2. **统一接口**：所有数据源实现相同的接口，上层调用无需关心底层差异
3. **来源追踪**：使用 Trace 机制精确追踪数据来源，支持多源数据关联
4. **缓存优先**：资源获取优先使用缓存，减少重复网络请求
5. **平台无关**：不硬编码任何具体平台名称，符合接口规范即可接入

## 核心架构

### 1. 源管理器（SourceManager）

`SourceManager` 是整个网络仓库的核心，负责：

- 源类型注册与管理
- 数据源生命周期管理
- Trace 导航与资源获取
- 多源搜索与结果聚合
- 默认源设置与切换

**主要功能：**

```javascript
// 注册源类型工厂
registerSourceType(type, factory)

// 添加数据源
addSource(sourceId, sourceName, type, options)

// 获取数据源
getSource(sourceId)

// 根据 Trace 获取资源
fetchResourceByTrace(trace, options)

// 多源搜索
searchAllSources(keyword, options)
```

### 2. 源基类（Source）

`Source` 是所有数据源的抽象基类，定义了统一的接口规范：

**数据操作接口：**
- `getMusicList()` - 获取音乐列表
- `getAlbums()`, `getAlbumById()`, `getAlbumsSongsById()` - 专辑相关操作
- `getArtists()`, `getArtistById()`, `getArtistsSongsById()` - 艺术家相关操作

**资源获取接口：**
- `getAlbumCover(albumId, maxResolution)` - 获取专辑封面
- `getMusicFile(songId)` - 获取音乐文件

**搜索接口：**
- `searchTracks()`, `searchAlbums()`, `searchArtists()`, `searchAll()` - 搜索功能

### 3. 内置源类型

#### 3.1 TauriSource（本地文件源）

- 类型：`tauri`
- 特点：通过 Tauri IPC 与 Rust 后端通信
- 功能：本地文件扫描、音乐标签读取、专辑封面提取

#### 3.2 WebSource / ApiSource（网络 API 源）

- 类型：`web` / `api`
- 特点：通过 HTTP 请求与远程 API 通信
- 功能：搜索、歌单、播放等

#### 3.3 NeteaseSource（网易云音乐源）

- 类型：`netease`
- 特点：专门适配网易云音乐 API
- 功能：完整的网易云音乐功能支持

### 4. Trace 来源追踪机制

`Trace` 是数据来源的核心追踪对象，记录：

- `sourceType` - 来源类型（Storage/Api）
- `sourceId` - 来源唯一标识
- `sourceName` - 来源显示名称
- `dataType` - 数据类型（Track/Artist/Album/Playlist）
- `dataId` - 数据 ID
- `fetchMethod` - 资源获取方法

**Trace 类型：**
- 本地文件 Trace：`Trace.createLocalFile()`
- API 来源 Trace：`Trace.createApiSource()`
- WebDAV Trace：`Trace.createWebDAV()`

## 使用示例

### 初始化源管理器

```javascript
import { sourceManager } from '@/api/source';

// 源管理器已默认注册本地源
const localSource = sourceManager.getDefaultSource();
```

### 添加网络 API 源

```javascript
// 添加自定义 API 源
const apiSource = sourceManager.addSource(
    'my-api',
    '我的音乐源',
    'api',
    { baseUrl: 'https://api.example.com' }
);
```

### 数据操作

```javascript
// 获取音乐列表
const tracks = await sourceManager.getMusicList();

// 获取专辑
const albums = await sourceManager.getAlbums();
const album = await sourceManager.getAlbumById(albumId);

// 获取专辑歌曲
const albumSongs = await sourceManager.getAlbumsSongsById(albumId);
```

### 资源获取

```javascript
import { Track } from '@/api/resources';

// 通过 Trace 获取资源
const track = Track.fromRaw(trackData);
const trace = track.primaryTrace;
const result = await sourceManager.fetchResourceByTrace(trace);

// result: { objectURL, destroyObjectURL, fromCache }
```

### 多源搜索

```javascript
// 在所有源中搜索
const results = await sourceManager.searchAllSources('周杰伦', {
    includeLocal: true
});

// results: { tracks, albums, artists, sources }
```

## 扩展开发

### 自定义数据源

1. 继承 `Source` 基类
2. 实现所有必需的接口方法
3. 注册源类型工厂

```javascript
import { Source, SOURCE_TYPE } from './base.js';

class CustomSource extends Source {
    constructor(sourceId, sourceName, baseUrl) {
        super(sourceId, sourceName, 'custom', SOURCE_TYPE.API);
        this.baseUrl = baseUrl;
    }

    async getMusicList() {
        // 实现获取音乐列表
    }

    // 实现其他接口...
}

// 注册源类型
sourceManager.registerSourceType('custom', (sourceId, sourceName, options) => {
    return new CustomSource(sourceId, sourceName, options.baseUrl);
});
```

## 架构优势

1. **可扩展性强**：新增数据源只需实现接口并注册
2. **统一调用**：上层代码无需关心数据源差异
3. **多源融合**：支持同时从多个源获取和聚合数据
4. **缓存优化**：统一的资源缓存机制
5. **来源可追溯**：每个数据都有完整的来源信息
