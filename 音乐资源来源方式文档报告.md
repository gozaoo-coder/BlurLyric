# 音乐资源来源方式文档报告

## 1. 概述

本报告详细介绍了音乐资源的各种来源方式及其实现机制，包括本地音乐库、网络音乐 API 等多种数据源的集成与管理方案。

## 2. 核心架构

### 2.1 整体架构

系统采用**多源管理**架构，通过统一的 `SourceManager` 管理多种类型的音乐数据源：

```
┌─────────────────────────────────────────────────────────┐
│                     SourceManager                      │
├───────────────────┬───────────────────┬─────────────────┤
│  TauriSource      │   WebSource       │  NeteaseSource  │
│  (本地音乐库)     │   (通用网络 API)  │  (网易云音乐)   │
├───────────────────┼───────────────────┼─────────────────┤
│  本地文件系统     │  自定义 Web API   │  网易云音乐 API │
└───────────────────┴───────────────────┴─────────────────┘
```

### 2.2 核心组件

- **SourceManager**：统一管理所有数据源，提供多源注册、资源获取、搜索等功能
- **Source**：抽象基类，定义数据源的通用接口
- **具体实现**：TauriSource、WebSource、NeteaseSource 等
- **Trace**：来源追踪系统，实现跨源数据关联
- **ResourceFetcher**：资源获取器，处理缓存策略

## 3. 数据源类型

### 3.1 本地音乐库（TauriSource）

**特点**：
- 基于 Tauri 后端，直接访问本地文件系统
- 支持增量扫描和全量扫描
- 多级缓存策略（内存 + 磁盘）
- 按需加载机制

**实现细节**：
- 使用 Tauri 命令调用 Rust 后端
- 支持音乐文件的增删改查
- 自动解析音乐元数据
- 支持专辑封面的懒加载

**核心功能**：
- 音乐文件管理（添加/移除音乐目录）
- 音乐库扫描（增量/全量）
- 本地缓存管理
- 性能监控

### 3.2 网络音乐 API（WebSource）

**特点**：
- 通用 Web API 适配器
- 支持自定义 API 地址
- 内置缓存机制
- 统一的数据解析接口

**实现细节**：
- 基于 fetch API 调用网络接口
- 支持搜索、详情获取、资源下载
- 可扩展为特定平台的实现

**核心功能**：
- 歌曲/专辑/艺术家搜索
- 详情信息获取
- 音频资源 URL 获取
- 专辑封面获取

### 3.3 网易云音乐（NeteaseSource）

**特点**：
- 基于 NeteaseCloudMusicApiEnhanced 项目
- 支持无损音质
- 完整的网易云音乐 API 映射
- 智能音质选择

**实现细节**：
- 实现网易云音乐的完整 API 端点
- 支持音质切换（标准/无损）
- 歌单功能支持
- 封面图片尺寸处理

**核心功能**：
- 网易云音乐搜索
- 歌单管理
- 无损音质支持
- 歌词获取

## 4. 资源获取流程

### 4.1 统一资源获取流程

```
sourceManager.fetchResourceByTrace(trace) →
  1. 检查 trace 类型（本地/远程）
  2. 本地资源：直接读取文件
  3. 远程资源：
     a. 检查缓存 → 命中则返回缓存
     b. 未命中则从网络下载
     c. 存入临时缓存池
     d. 返回 ObjectURL
```

### 4.2 缓存策略

- **本地资源**：直接访问文件系统，无需缓存
- **远程资源**：
  - 内存缓存（API 响应）
  - 临时文件缓存（音频文件）
  - 图片缓存（专辑封面）

### 4.3 预加载机制

- 支持单个资源预加载
- 支持批量资源预加载（并发控制）
- 后台异步下载，提升用户体验

## 5. 扩展机制

### 5.1 数据源扩展

系统采用**工厂模式**注册新的数据源类型：

```javascript
// 注册新的数据源类型
sourceManager.registerSourceType('custom', (sourceId, sourceName, options) => {
  return new CustomSource(sourceId, sourceName, options);
});

// 添加数据源实例
sourceManager.addSource('mycustom', '自定义数据源', 'custom', {
  apiUrl: 'https://api.example.com'
});
```

### 5.2 数据类型扩展

通过 Trace 系统支持新的数据类型：

- 歌曲（TRACK）
- 专辑（ALBUM）
- 艺术家（ARTIST）
- 歌单（PLAYLIST）
- 可扩展自定义数据类型

## 6. 使用示例

### 6.1 基本使用

```javascript
// 获取默认数据源（本地音乐库）
const localSource = sourceManager.getDefaultSource();

// 获取音乐列表
const musicList = await localSource.getMusicList();

// 搜索歌曲
const searchResult = await sourceManager.searchAllSources('周杰伦', {
  includeLocal: true
});

// 获取音乐文件
const musicFile = await sourceManager.getMusicFile('123456');
```

### 6.2 多源搜索

```javascript
// 在多个数据源中搜索
const results = await sourceManager.searchAllSources('周杰伦', {
  sourceIds: ['local', 'netease'], // 指定数据源
  includeLocal: true
});

// 结果包含每个源的搜索结果
console.log(results.sources);
console.log(results.tracks);
```

### 6.3 资源预加载

```javascript
// 预加载歌曲资源
const trace = sourceManager.createLocalTrace(
  TraceDataType.TRACK,
  '123456',
  '/path/to/music.mp3'
);

await sourceManager.preloadResource(trace);

// 批量预加载
const traces = [
  // 多个 trace 对象
];
await sourceManager.preloadResources(traces, 3); // 并发数 3
```

## 7. 性能优化

### 7.1 缓存优化

- **多级缓存**：内存缓存 + 磁盘缓存
- **增量扫描**：只扫描新增或修改的文件
- **按需加载**：仅在需要时加载资源
- **缓存过期策略**：自动清理过期缓存

### 7.2 网络优化

- **并发控制**：限制并发请求数
- **错误重试**：网络失败自动重试
- **智能音质选择**：根据网络状况选择合适的音质
- **批量请求**：合并多个请求减少网络开销

### 7.3 内存管理

- **ObjectURL 管理**：使用后及时销毁
- **缓存大小限制**：防止内存溢出
- **垃圾回收**：定期清理无用缓存

未来，随着音乐服务的不断发展，本系统可以通过扩展新的数据源和功能，为用户提供更加丰富和个性化的音乐体验。
